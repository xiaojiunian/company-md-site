## 公司新闻

这里可以放公司近期的重要动态，例如新产品发布、服务升级、团队活动、合作伙伴进展等。

## 产品更新

- 官网内容支持 Markdown 维护
- 页面可自动转换为 HTML
- GitHub Pages 发布后会自动更新线上地址
- 后续可以继续扩展多页面导航和内容分类

## 行业观察

企业官网不一定要一开始就做得很复杂。对于介绍页、产品说明页、案例页和新闻页，先用 Markdown 建立稳定的内容结构，再统一套用设计样式，是一种轻量但可持续的方式。

## 后续可以怎么扩展

如果新闻内容变多，可以继续拆成这样的结构：

- `pages/news/company-update.md`
- `pages/news/product-release.md`
- `pages/news/customer-story.md`

每一个 Markdown 文件都可以对应一个页面地址。
